# Android on-device

Script: `scripts/android-device-build.sh`. Builds the preview APK and installs
it in place on a connected device.

## First-time machine setup

1. **platform-tools (adb):** install the Android command-line tools /
   platform-tools. Owner (macOS/Homebrew):
   `/opt/homebrew/share/android-commandlinetools/platform-tools`. Put
   `platform-tools` on PATH (the script does this via `ANDROID_HOME`).
2. **Java 21:** newer JDKs (26) break the gradle template. macOS:
   `JAVA_HOME=$(/usr/libexec/java_home -v 21)`.
3. **Enable USB debugging on the phone:** Settings → About → tap Build number
   7× → Developer options → USB debugging on.
4. **Authorize the device:** plug in, tap "Allow" on the RSA-key prompt, then:
   ```bash
   adb devices          # should show your device as "device" (not "unauthorized")
   ```
   Stale daemon / "unauthorized" that won't clear: `adb kill-server && adb devices`.
5. Set `DEVICE_MATCH` in the script to a substring of your device's
   `adb devices -l` line (owner: `model:Pixel_7`).

## Why the recipe is the way it is

- **Stable keystore:** the release APK signs with the debug keystore, which is
  gitignored and REGENERATED by every `expo prebuild --clean` (new random key).
  A changed key → `INSTALL_FAILED_UPDATE_INCOMPATIBLE` → forced uninstall (wipes
  app data). The persistent key lives OUTSIDE the repo
  (`~/.config/t3code-android/debug.keystore`, machine-local by owner choice, NOT
  committed); the script copies it in after prebuild so updates install in
  place. First run adopts whatever prebuild generated as the durable key. If the
  key is ever lost, the next install needs one uninstall+wipe, then it
  re-stabilizes.
- **vp node vs system node:** the system/nix `node` SIGABRTs (exit 134) inside
  Metro bundling on the owner's Mac. `~/.vite-plus/bin` MUST lead PATH.
- **`gradlew --stop` first:** kills stale gradle daemons that otherwise cause
  flaky/OOM builds. Heap is bumped via `-Dorg.gradle.jvmargs=-Xmx6g`.

Install-only when the device is absent: the script prints the APK path; run
`adb install -r <apk>` once plugged in.
